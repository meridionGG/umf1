## 1. Архитектура и структуры данных (`fn.h`)

В этом заголовочном файле мы объявляем всё, из чего состоит программа. C++ требует строгого описания типов данных, поэтому мы используем `struct` для группировки логически связанных переменных.

### Описание области и сетки
```cpp
struct area_element_t {
    int func_number;
    int x1, y1, x2, y2;
};
```
Сложная Т-образная форма не может быть задана одной границей. Мы разбиваем её на **прямоугольные блоки** (элементы). `x1, y1, x2, y2` — это индексы координатных линий, которые ограничивают этот прямоугольный блок.

```cpp
struct area_t {
    std::vector<double> x_lines, y_lines; 
    std::vector<int> x_indices, y_indices; 
    std::vector<area_element_t> elems; 
};
struct grid_t {
    std::vector<double> x, y; 
};
```
- `area_t` хранит **макро-геометрию**: главные линии, образующие "каркас" буквы Т.
- `grid_t` хранит **микро-геометрию**: точные координаты $x$ и $y$ для каждого из тысяч узлов расчетной сетки, сгенерированной внутри этого каркаса.

### Структура матрицы СЛАУ
```cpp
struct slae_t {
    std::vector<double> u1, u2, di, l4, l5, b;
    std::vector<int> ig;
};
```
Здесь скрыта вся магия экономии памяти. При сетке $100 \times 100$ узлов, обычная матрица занимала бы $10^8$ элементов (сотни мегабайт). Но в МКР каждое уравнение связывает узел только с его 4 соседями (слева, справа, сверху, снизу) и самим собой.
Мы храним только 5 диагоналей:
- `di`: главная диагональ (сам узел).
- `u2`, `l4`: верхняя и нижняя диагонали (соседи по X).
- `u1`, `l5`: дальние верхняя и нижняя диагонали (соседи по Y, они отстоят на `n_x` элементов).
- `b`: вектор правой части (значения функции $f(x,y)$).

### Аналитические функции для тестов
```cpp
struct functionsBC {
    std::function<double(double, double)> u_exact;
    std::function<double(double, double, parameters_t)> f_rhs;
    std::vector<std::function<double(double, double)>> firstBC;
    std::vector<std::function<double(double, double)>> thirdBC;
};
```
В C++11 появилась удобная обертка `std::function`. Эта структура позволяет передавать математические функции как переменные. Мы можем "положить" в `u_exact` функцию $u = x^2 + y^2$, и программа будет её использовать как точное решение.

## 2. Чтение данных и построение сетки (`input.cpp`)

Файл отвечает за загрузку входных файлов из папки `in/`.

### Построение сетки с геометрической прогрессией (`read_grid`)
Самая сложная часть — разбиение макро-области на мелкую сетку. Сетка может быть неравномерной (чтобы сгущать узлы там, где решение резко меняется).
```cpp
if (q != 1) {
    double sumProgression = (pow(q, countInterval) - 1.0) / (q - 1.0);
    step = (area.x_lines[i + 1] - area.x_lines[i]) / sumProgression;
    // ...
}
```
Если коэффициент `q` не равен 1, расстояние между узлами изменяется по закону геометрической прогрессии. Формула `sumProgression` вычисляет сумму этой прогрессии, чтобы гарантировать, что последний узел точно попадет на правую границу `area.x_lines[i + 1]`.

## 3. Ядро метода конечных разностей (`fdm.cpp`)

Именно здесь дифференциальное уравнение превращается в СЛАУ.

### Фиктивные узлы
```cpp
bool isDummyNode(area_t &area, int i, int j)
```
Функция проверяет, принадлежит ли узел с индексами `(i, j)` Т-образной области. Если узел лежит вне всех прямоугольных блоков `area.elems`, он считается "фиктивным" и возвращает `true`.

### Сборка матрицы МКР (`createSLAE`)
```cpp
slae.di[k] = params.lambda * (2.0 / (hx1 * hx2) + 2.0 / (hy1 * hy2)) + params.gamma;
slae.u2[k - 1] = -params.lambda * 2.0 / (hx1 * (hx1 + hx2));   // левый
slae.l4[k]     = -params.lambda * 2.0 / (hx2 * (hx1 + hx2));   // правый
```
Это реализация второй производной на **неравномерной** сетке. 
- `hx1` — шаг слева, `hx2` — шаг справа.
- Если сетка равномерная ($hx1 = hx2 = h$), то формула `2.0 / (hx1 * (hx1 + hx2))` магическим образом превращается в классическое $1 / h^2$.

### Обработка краевых условий (`applyBoundaryConds`)
Здесь уравнения в узлах на границе заменяются на новые.
```cpp
// Краевое условие 1-го рода (Дирихле)
slae.di[k] = 1.0;
// обнуление связей с соседями
if (k - n_x >= 0) slae.u1[k - n_x] = 0; ... 
slae.b[k] = func.firstBC[0](x, y);
```
Для условия Дирихле нам нужно уравнение $1 \cdot u_i = g$. Поэтому мы ставим `1.0` на главную диагональ, нули на соседние диагонали, и точное значение в правую часть `b`.

Условия **Робена (3-го рода)** обрабатываются сложнее — там используется аппроксимация 2-го порядка точности, в которую вшиваются шаги сетки `hx` и `hy`, чтобы не терять точность всего метода на границе.

## 4. Решатель СЛАУ: Метод релаксации и прогонка (`sor.cpp`)

После генерации матрицы мы имеем огромную СЛАУ ($Au = b$). Функция `method_BlockRelaxation` решает её итерационно.

### Алгоритм Томаса (Метод прогонки)
```cpp
void tridiagonal_solve(const std::vector<double>& a, const std::vector<double>& c, 
                       const std::vector<double>& b, const std::vector<double>& f, std::vector<double>& x)
```
Это супер-быстрый метод Гаусса, адаптированный специально для трехдиагональных матриц. Он имеет прямой ход (вычисление коэффициентов `alpha` и `beta`) и обратный ход (поиск `x[i]`). Выполняется мгновенно.

### Блочная релаксация по строкам
Вместо того чтобы решать всю 2D-систему сразу, алгоритм решает её строками (горизонтальными линиями).
```cpp
for (int j = 0; j < n_y; ++j) {
    // ...
    // f[i] = b[k] - влияние верхней строки - влияние нижней строки
    if (j > 0) f[i] -= slae.u1[k - n_x] * u[k - n_x];
    if (j < n_y - 1) f[i] -= slae.l5[k] * uk[k + n_x]; 
    
    tridiagonal_solve(a, c, b_sub, f, x_line);
}
```
Логика:
1. Замораживаем значения функции на соседних горизонтальных линиях.
2. Переносим эти известные значения в правую часть уравнения `f[i]`.
3. Оставшаяся матрица для текущей строки $j$ становится строго трехдиагональной (связи только "влево-вправо").
4. Решаем её методом прогонки `tridiagonal_solve`.
5. Повторяем процесс, пока суммарная невязка не станет меньше $10^{-13}$ (`eps`).

## 5. Точка входа и тестирование сходимости (`main.cpp`)

Функция `main` запускает процесс тестирования, чтобы доказать математическую корректность кода.

### Подготовка тестовых функций
Для тестирования мы задаем функцию принудительно:
```cpp
functionsBC f_quad;
f_quad.u_exact = [](double x, double y) { return x * x + y * y; };
f_quad.f_rhs = [](double x, double y, parameters_t p) { return -p.lambda * 4.0 + p.gamma * (x * x + y * y); };
```
Здесь $u(x,y) = x^2 + y^2$. 
Оператор Лапласа (сумма вторых производных) для нее равен $2 + 2 = 4$. 
Поэтому мы аналитически пишем правую часть: $- \lambda \cdot 4 + \gamma \cdot (x^2 + y^2)$. Таким образом, подставляя `f_rhs` в метод конечных разностей, он обязан на выходе выдать функцию $x^2 + y^2$.

### Цикл исследования сходимости
Функция `run_convergence_study` запускает решатель для сеток $N=10$, $N=20$, $N=40$.

```cpp
double p = log2(errs[i - 1] / errs[i]);
```
- `errs[i-1]` — максимальная ошибка на старой сетке (например, 10).
- `errs[i]` — максимальная ошибка на новой, более густой сетке (20).
Если метод реализован правильно (без багов и с правильными краевыми условиями), ошибка на сетке в 2 раза гуще должна упасть примерно в $2^2 = 4$ раза. Логарифм по основанию 2 от 4 равен 2. Значит, выведенный порядок сходимости `p` должен стремиться к двойке. 

**Если `p` в выводе программы равно $\approx 2.0$, это означает абсолютную победу и то, что программа написана безупречно.**
